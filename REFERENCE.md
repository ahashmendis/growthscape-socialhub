# Primary Product Reference

## Metricool

Official Website

https://metricool.com/

Metricool is the **primary product, workflow, and UX reference** for Growthscape Social Hub.

Before implementing **every major feature**, always visit the live Metricool website and application to understand how the feature currently works.

Never rely solely on previous knowledge or memory.

Always research the current implementation.

Study the product as if performing a UX audit.

## Before Implementing Any Feature

Answer these questions:

* What problem is Metricool solving?
* Why is the workflow designed this way?
* What information is most important?
* How many clicks does the user need?
* What makes the workflow intuitive?
* What are the strengths?
* What are the weaknesses?
* How could AI improve this experience?
* How could automation improve this experience?
* How can Growthscape Social Hub reduce manual work?

Always understand the user experience before writing code.

# UI Guidelines

The application should feel:

* Modern
* Fast
* Premium
* AI-first
* Professional
* Clean
* Minimal
* Production-ready

Avoid:

* Generic admin templates
* Basic CRUD layouts
* Bootstrap-looking interfaces
* Poor spacing
* Default HTML styling
* Placeholder pages
* Inconsistent components

# UX Principles

Prioritize:

* Minimal clicks
* Clear navigation
* Excellent information hierarchy
* Consistent interactions
* Fast workflows
* Responsive layouts
* Accessibility
* Helpful empty states
* Beautiful loading states

# Dashboard Philosophy

The dashboard should always answer:

1. What happened?
2. Why did it happen?
3. What should I do next?
4. What will likely happen next?

Metricool mainly answers the first two.

Growthscape should answer all four.

---

# AI Philosophy

AI is the biggest differentiator.

Every analytics page should provide:

* Explanation
* Recommendation
* Prediction
* Suggested Action

Do not simply display numbers.

Always help the user make better decisions.

# Research Workflow

Whenever implementing a new feature:

1. Read the project documentation.
2. Visit the live Metricool website and application.
3. Research the equivalent feature.
4. Analyze the workflow.
5. Identify strengths and weaknesses.
6. Design an improved Growthscape version.
7. Implement using the project's architecture and design system.
8. Test thoroughly.
9. Document the implementation.

Never skip the research phase.
